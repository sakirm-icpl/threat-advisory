Getting Started with Version Detection

Learn how to contribute to the open-source version detection pattern database

Welcome to the Community!

Thank you for your interest in contributing to the Version Detection Pattern Database. This guide will help you understand how to contribute regex patterns for software version detection.

1 Understand the Project

Our database contains regex patterns that help identify software versions from network responses. These patterns are used by security professionals for:

  • Penetration testing
  • Vulnerability assessment
  • Bug bounty hunting
  • Asset discovery
  • Security scanning tools

2 Explore Existing Patterns

Before creating new patterns, explore our existing database to understand the structure:

  • View patterns in the Pattern Database
  • Check the patterns directory on GitHub
  • Review the Pattern Template

3 Set Up Your Environment

To contribute patterns, you'll need:

  • A GitHub account
  • Git installed on your system
  • Python 3.6+ (for validation tools)
  • A text editor or IDE

Install the required dependencies:

pip install jsonschema

4 Find a Service to Pattern

Look for services that disclose version information. Some ideas:

  • Web servers (Apache, Nginx, IIS)
  • Database systems (MySQL, PostgreSQL, MongoDB)
  • Networking services (SSH, FTP, SMTP)
  • Applications (WordPress, Drupal, Jenkins)
  • Messaging systems (Kafka, RabbitMQ)

5 Create Your Pattern

Follow these steps to create a new pattern:

  1. Fork the repository
  2. Create a new branch: git checkout -b pattern/service-name
  3. Copy the pattern template
  4. Create your regex pattern with a version capture group
  5. Add comprehensive test cases
  6. Validate with our tools

6 Validate Your Pattern

Use our validation script to ensure your pattern is correct:

python tools/validate-pattern.py patterns/web/your-pattern.json

This checks:

  • JSON schema compliance
  • Regex compilation
  • Test case execution

7 Submit Your Contribution

When your pattern is ready:

  1. Commit your changes
  2. Push to your fork
  3. Create a pull request
  4. Respond to feedback during review

Need Help?
