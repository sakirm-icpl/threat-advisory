Beginner's Guide to Contributing

Welcome to the Threat Advisory Database! This guide will help you get started contributing to our open-source project, even if you're new to cybersecurity or open-source development.

What is this project?

The Threat Advisory Database is a collection of threat intelligence advisories and vulnerability data used to inform security professionals about emerging threats. These advisories help organizations protect themselves by providing timely information about security risks.

How to Contribute

Step 1: Fork the Repository

Start by forking our repository on GitHub. This creates your own copy where you can make changes without affecting the main project.

  1. Visit our GitHub repository
  2. Click the "Fork" button in the top right corner
  3. Choose your GitHub account as the destination for the fork

Step 2: Clone Your Fork

Clone your forked repository to your local machine:

git clone https://github.com/YOUR-USERNAME/threat-advisory.git
cd threat-advisory

Step 3: Create a New Branch

Create a new branch for your work to keep changes organized:

git checkout -b add-my-new-advisory

Step 4: Research Your Advisory

Before creating an advisory, conduct thorough research:

  • Verify the threat or vulnerability from multiple sources
  • Gather evidence and proof of concept where possible
  • Identify affected products and versions
  • Determine the potential impact
  • Research existing advisories to avoid duplication

Step 5: Create Your Advisory File

Create a new advisory file in the appropriate category directory:

  1. Navigate to the advisories/ directory
  2. Choose the appropriate subdirectory (cve, malware, apt, etc.)
  3. Create a new JSON file using the advisory ID (e.g., cve-2024-12345.json)
  4. Use our advisory template as a starting point

Step 6: Add References and Recommendations

Every advisory must include references and recommendations:

  • Add authoritative references to sources
  • Include actionable recommendations for protection
  • Provide reliable URLs for further information

Step 7: Validate Your Advisory

Use our validation tool to check your advisory:

python tools/validate-advisory.py advisories/cve/cve-2024-12345.json

Fix any issues that the validation tool reports.

Step 8: Commit and Push Your Changes

git add advisories/cve/cve-2024-12345.json
git commit -m "Add advisory for CVE-2024-12345"
git push origin add-my-new-advisory

Step 9: Create a Pull Request

Go to your fork on GitHub and create a pull request to merge your changes into the main repository.

Best Practices

Research Thoroughly

Spend time verifying information from multiple authoritative sources before creating an advisory.

Be Specific

Make your advisories specific enough to be actionable but comprehensive enough to be useful.

Provide Actionable Recommendations

Include clear, practical recommendations that organizations can implement to protect themselves.

Follow the Template

Use our standard advisory template and fill out all required fields with accurate information.

Need Help?

If you need assistance:

  • Check our contribution guidelines
  • Look at existing advisories as examples
  • Open an issue on GitHub with your question
  • Join our community discussions
