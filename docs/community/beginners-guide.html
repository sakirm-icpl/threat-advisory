Beginner's Guide to Contributing

Your step-by-step introduction to contributing to the Version Detection Pattern Database

Welcome to the Community!

If you're new to open-source contribution or cybersecurity research, this guide is for you. We've designed a simple, step-by-step process to help you contribute your first version detection pattern to our database.

üí° First Time Contributor?

If you're new to GitHub, we recommend starting with GitHub's Hello World guide to familiarize yourself with the basics of creating repositories, branches, commits, and pull requests.

What You'll Learn

By following this guide, you'll learn how to:

  • Understand what version detection patterns are
  • Find services that disclose version information
  • Create regex patterns to detect software versions
  • Write test cases to validate your patterns
  • Submit your pattern to the database

What Are Version Detection Patterns?

Version detection patterns are regular expressions (regex) that identify software versions from network responses. For example, when a web server responds to an HTTP request, it might include a "Server" header like:

Server: Apache/2.4.41 (Ubuntu)

A version detection pattern for this would be a regex that can extract "2.4.41" from this response. These patterns are used by security professionals for:

  • Penetration testing
  • Vulnerability assessment
  • Bug bounty hunting
  • Asset discovery

Step-by-Step Contribution Process

1 Set Up Your Environment

Before you start, you'll need:

  • A GitHub account (free to create)
  • Git installed on your computer
  • A text editor (VS Code, Sublime Text, etc.)
  • Python 3.6+ (for validation tools)

Install the required Python package:

pip install jsonschema

2 Find a Service to Research

Look for services that disclose version information in their responses. Some common examples include:

  • Web servers: Apache, Nginx, IIS
  • Database systems: MySQL, PostgreSQL, MongoDB
  • Networking services: SSH, FTP, SMTP
  • Applications: WordPress, Jenkins, Tomcat

You can find these by:

  • Scanning your own test environments
  • Researching publicly available documentation
  • Looking at existing patterns in our database to find gaps

3 Fork and Clone the Repository

Visit our GitHub repository and click the "Fork" button in the top-right corner. This creates your own copy of the repository.

Clone your fork to your local machine:

git clone https://github.com/YOUR-USERNAME/version-detection-db.git
cd version-detection-db

4 Create a New Branch

Create a new branch for your pattern:

git checkout -b pattern/service-name

Replace "service-name" with the name of the service you're creating a pattern for (e.g., "pattern/apache" or "pattern/mysql").

5 Create Your Pattern File

Create a new JSON file in the appropriate category directory:

  • patterns/web/ - For web servers and applications
  • patterns/networking/ - For networking services
  • patterns/database/ - For database systems
  • patterns/messaging/ - For messaging systems

    • Copy the template to get started:

    cp patterns/TEMPLATE.md patterns/web/your-service.json

    For example, if you're creating a pattern for Nginx:

    cp patterns/TEMPLATE.md patterns/web/nginx.json

6 Write Your Pattern

Open your new file and replace the template content with your pattern information:

Key fields to update:

  • name: Descriptive name (e.g., "Nginx Web Server")
  • category: One of web, networking, database, messaging
  • pattern: Your regex pattern with version capture group
  • vendor: Software vendor (e.g., "Nginx Inc.")
  • product: Software product name (e.g., "Nginx")
  • version_group: Capture group number for version (usually 1)
  • priority: 1-999 (higher = more important)
  • confidence: 0.0-1.0 (how reliable is your pattern?)

In the metadata section:

  • author: Your name or GitHub username
  • created_at: Current date in ISO format
  • description: Explain what your pattern detects
  • tags: Relevant keywords for your pattern

7 Add Test Cases

Add at least 2-3 test cases in the test_cases array:

"test_cases": [
  {
    "input": "Server: nginx/1.18.0 (Ubuntu)",
    "expected_version": "1.18.0"
  },
  {
    "input": "Server: nginx/1.20.1",
    "expected_version": "1.20.1"
  }
]

Your test cases should include:

  • Real examples from actual service responses
  • Variations in formatting (if applicable)
  • Edge cases that might cause false positives

8 Validate Your Pattern

Run the validation script to check your pattern:

python tools/validate-pattern.py patterns/web/your-file.json

This will check:

  • JSON schema compliance
  • Regex compilation
  • Test case execution

Fix any errors that appear.

9 Commit and Push Your Changes

Add your new file to git:

git add patterns/web/your-file.json
git commit -m "Add pattern for Nginx version detection"

Push to your fork:

git push origin pattern/service-name

10 Create a Pull Request

Visit your fork on GitHub and create a pull request:

  1. Click "Compare & pull request"
  2. Write a descriptive title and explanation
  3. Submit the pull request

Maintainers will review your pattern and may suggest improvements.

Example Pattern

Here's a complete example for Nginx:

{
  "patterns": [
    {
      "name": "Nginx Web Server",
      "category": "web",
      "pattern": "Server: nginx/([\\d.]+)",
      "vendor": "Nginx Inc.",
      "product": "Nginx",
      "version_group": 1,
      "priority": 170,
      "confidence": 0.95,
      "metadata": {
        "author": "Your Name",
        "created_at": "2025-09-30T10:00:00",
        "updated_at": "2025-09-30T10:00:00",
        "description": "Detects Nginx version from HTTP server banner. This pattern matches the Server header commonly returned by Nginx servers.",
        "tags": ["http", "nginx", "webserver"],
        "test_cases": [
          {
            "input": "Server: nginx/1.18.0 (Ubuntu)",
            "expected_version": "1.18.0"
          },
          {
            "input": "Server: nginx/1.20.1",
            "expected_version": "1.20.1"
          }
        ]
      }
    }
  ]
}

Tips for Success

üí° Regex Tips

  • Use capture groups () around the version part you want to extract
  • Escape backslashes in JSON strings (use \\d instead of \d)
  • Be specific to avoid false positives
  • Test your pattern against various examples

üí° Testing Tips

  • Include real examples from actual services
  • Test edge cases and variations
  • Check for false positives with similar strings
  • Validate with the provided tool before submitting
