Beginner-friendly issues to help you start contributing to the Version Detection Pattern Database

These issues are specifically selected for newcomers to the project. They're relatively straightforward to implement and don't require deep knowledge of the entire codebase. Each issue includes clear instructions and is a great way to get familiar with our contribution process.

Create Pattern for Redis Database

We need a version detection pattern for Redis database servers. Redis typically discloses version information in its initial connection response.

Beginner Pattern Database

Details

Redis servers respond with version information when a client connects. The response looks like:

+OK
$4
redis_version:6.2.6

Your task is to create a regex pattern that can extract the version number (e.g., "6.2.6") from this response.

Requirements

  • Create a pattern file in patterns/database/redis.json
  • Follow the pattern template structure
  • Include at least 3 test cases with real examples
  • Validate with our tools before submitting

Add Test Cases to Existing Patterns

Many existing patterns could benefit from additional test cases to improve their reliability and coverage.

Beginner Testing Pattern

Details

Review existing patterns in the database and add additional test cases to improve their coverage. Look for patterns with fewer than 3 test cases or patterns that might have edge cases not currently covered.

Requirements

  • Identify patterns that need more test cases
  • Research real-world examples of the service responses
  • Add 2-3 additional test cases to each selected pattern
  • Ensure all test cases pass validation

How to Contribute

1 Choose an Issue

Browse the issues above and select one that interests you. If you're new to the project, we recommend starting with a pattern creation issue.

2 Comment on the Issue

Before you start working, comment on the GitHub issue to let others know you're working on it. This prevents duplicate work.

3 Follow Our Contribution Process

Follow the steps outlined in our Beginner's Guide:

  1. Fork the repository
  2. Create a new branch
  3. Make your changes
  4. Validate your work
  5. Submit a pull request

4 Get Feedback

Maintainers will review your pull request and may suggest improvements. Be responsive to feedback and make requested changes.

Need Help?

If you need assistance with any of these issues:

  • Ask questions in GitHub Discussions
  • Review existing issues and pull requests for examples
  • Contact maintainers directly for complex issues
