Good First Issues

Welcome! If you're new to contributing to open-source or to this project specifically, these issues are a great place to start. They're well-defined, relatively self-contained, and don't require deep knowledge of the entire codebase.

Pattern Creation

Easy

Add Pattern for Apache Tomcat

Create a regex pattern to detect Apache Tomcat versions from HTTP server headers or other common sources.

Skills needed: Basic regex knowledge, research skills

Expected outcome: A new pattern file in patterns/web/ with comprehensive test cases

Easy

Add Pattern for Redis

Create a regex pattern to detect Redis versions from server banners or protocol responses.

Skills needed: Basic regex knowledge, research skills

Expected outcome: A new pattern file in patterns/database/ with comprehensive test cases

Medium

Add Pattern for MongoDB

Create regex patterns to detect MongoDB versions from various sources like error messages, banners, or protocol responses.

Skills needed: Analytical skills, research skills

How to Get Started

  1. Read our Beginner's Guide if you haven't already
  2. Choose an issue that interests you and matches your skill level
  3. Fork the repository and create a new branch for your work
  4. Comment on the issue to let others know you're working on it
  5. Make your changes following our contribution guidelines
  6. Submit a pull request when you're ready for review
