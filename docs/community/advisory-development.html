Advisory Development Guide

This guide is for experienced contributors who want to create advanced advisories or improve existing ones. It covers best practices, advanced techniques, and common pitfalls to avoid.

Advanced Advisory Design

Understanding Advisory Structure

Each advisory follows a standardized JSON structure that makes it easy to process programmatically. Understanding this structure is crucial for creating high-quality advisories.

Complex Advisory Structures

For complex threats that require multiple advisories, consider creating multiple entries in a single file:

{
  "advisories": [
    {
      "id": "THREAT-2024-001",
      "title": "Initial Access Vector for Campaign X",
      "category": "threat",
      // ... other fields ...
    },
    {
      "id": "THREAT-2024-002",
      "title": "Lateral Movement Techniques in Campaign X",
      "category": "threat",
      // ... other fields ...
    }
  ]
}

Research Best Practices

Source Verification

Always verify information from multiple authoritative sources:

  • Vendor security advisories
  • National vulnerability databases
  • Threat intelligence feeds
  • Security research publications
  • Incident response reports

Common Pitfalls

Avoid these common mistakes:

  • Creating advisories based on unverified information
  • Duplicating existing advisories
  • Providing vague or non-actionable recommendations
  • Using unreliable or broken reference links

Advanced Content Creation

Writing Effective Recommendations

Recommendations should be:

  • Actionable and specific
  • Prioritized by effectiveness
  • Applicable to different environments
  • Clear and concise

Creating Comprehensive References

Include references that:

  • Link to authoritative sources
  • Provide additional context
  • Are accessible and reliable
  • Cover different aspects of the threat

Severity and Impact Assessment

Severity Scoring Guidelines

Use these guidelines for consistent severity scoring:

  • Critical: Immediate threat with severe impact (CVSS 9.0-10.0)
  • High: Significant threat with major impact (CVSS 7.0-8.9)
  • Medium: Moderate threat with notable impact (CVSS 4.0-6.9)
  • Low: Minimal threat with limited impact (CVSS 0.1-3.9)

Working with the Validation Tools

Automated Validation

Use our validation tools to check your advisories:

# Validate a single advisory
python tools/validate-advisory.py advisories/cve/cve-2024-12345.json

# Validate all advisories
python tools/validate-all-advisories.py

Continuous Integration

All advisories are automatically validated when submitted via pull request. Make sure your advisories pass validation before submitting.

Collaboration and Review

Peer Review Process

All advisory submissions go through peer review:

  1. Automated validation checks
  2. Manual review by maintainers
  3. Verification of sources and references
  4. Feedback and revision requests
  5. Approval and merge

Incorporating Feedback

Be open to feedback and willing to iterate on your advisories. Reviewers may suggest improvements to accuracy, clarity, or completeness.

Advanced Contribution Opportunities

  • Develop new validation tools or enhance existing ones
  • Create advisory analysis tools to identify potential improvements
  • Build integrations with popular threat intelligence platforms
  • Research and document emerging threat patterns
