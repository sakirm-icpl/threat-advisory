Contributing to Version Detection

Learn how to contribute to the open-source version detection pattern database

We welcome contributions from the security research community! This document provides guidelines for contributing to our project.

Ways to Contribute

Pattern Contributions

Submit new regex patterns for version detection:

  • New patterns for services not yet in the database
  • Improved patterns with better accuracy
  • Patterns for new service categories

Code Contributions

Improve our tools and infrastructure:

  • Enhance validation scripts
  • Develop new testing tools
  • Improve website and documentation

Documentation Improvements

Help make our documentation better:

  • Clarify existing documentation
  • Add new guides and tutorials
  • Improve examples and templates

Community Support

Support the community:

  • Answer questions in discussions
  • Review pull requests
  • Participate in project planning

Contribution Process

1

Fork and Clone

Fork the repository and clone it to your local machine:

git clone https://github.com/your-username/version-detection-db.git
cd version-detection-db
2

Create a Branch

Create a new branch for your contribution:

git checkout -b pattern/new-service
3

Make Changes

Make your changes following our guidelines:

  • Follow the pattern template
  • Include comprehensive test cases
  • Update documentation as needed
4

Validate Your Changes

Validate your patterns with our tools:

python tools/validate-pattern.py patterns/web/your-pattern.json
5

Commit and Push

Commit your changes and push to your fork:

git add patterns/web/your-pattern.json
git commit -m "Add pattern for NewService version detection"
git push origin pattern/new-service
6

Create Pull Request

Create a pull request with a clear description of your changes.

Code of Conduct

By participating in this project, you agree to follow our Code of Conduct. Please treat all participants with respect and professionalism.

Getting Help

If you need help with your contribution:

  • Check the documentation in the docs directory
  • Review existing issues and pull requests
  • Ask questions in GitHub Discussions
  • Contact maintainers directly for complex issues
